# CrimsonSon 玩法设计

本文档说明 CrimsonSon 当前版本的玩法规则和主要流程。它面向服务器管理者、测试人员和参与维护的开发者，用于确认插件在回合中的预期行为。

## 阵营目标

深红之子是第三方阵营。阵营目标不是常规正面对抗，而是护送 `HolyFather`（教皇）进入 `SCP-049` 或 `SCP-106` 收容室，使用 `SCP1576` 完成深红献祭仪式。

仪式完成后，插件会处理非深红阵营的存活玩家并请求结束回合。最终结算显示由 SCP:SL 原生回合结算逻辑决定。

## 回合开始

回合开始后，插件会从当前 D 级人员中随机选择一名玩家成为 `SCP-999-B`。

`SCP-999-B` 会被切换为 `Tutorial` 角色，并传送到配置项 `Spawn999BRoom` 指定的房间，默认是 `LczGlassroom`。他的任务是进入广播室 `EzIntercom`，为深红之子的召唤创造条件。

如果回合开始时没有可用的 D 级人员，插件不会生成 `SCP-999-B`。

## 召唤阶段

`SCP-999-B` 进入广播室后会尝试召唤深红之子。

召唤限制：

- 每回合只允许成功召唤一次。
- 观察者人数必须达到 `MinimumSpectatorsToSummon`。
- 单次最多召唤 `MaximumSummonCount` 名观察者。

召唤成功后：

- 全服显示召唤成功提示。
- 触发 CASSIE 广播文案和配套音频。
- 播放召唤音效。
- 从观察者中生成深红之子成员。
- 优先保底生成 `HolyFather`、`Heathen`、`Hallow`。
- 剩余名额按角色权重分配，并受各角色人数上限限制。

如果观察者不足，插件会显示失败提示。只要当前 `SCP-999-B` 仍然有效、仍在广播室，并且回合仍在进行，就允许再次尝试召唤。

## 角色体系

深红阵营角色如下：

| ID | 角色 | 说明 |
|---|---|---|
| 0 | `SCP999B` | 开局被选中的特殊单位，用于触发召唤。 |
| 1 | `HolyFather` | 教皇，唯一可以启动仪式的核心角色。 |
| 2 | `Heathen` | 深红成员，负责保护教皇。 |
| 3 | `Hallow` | 深红成员，负责保护教皇。 |
| 4 | `Follower` | 深红成员，负责保护教皇。 |
| 5 | `EvilFollower` | 深红成员，负责保护教皇。 |
| 6 | `Fanatic` | 深红成员，负责保护教皇。 |

所有深红角色在游戏身份上都会使用 `Tutorial`。角色名称、介绍文本、人数上限、分配权重和血量可以通过配置或翻译文件调整；默认物品列表由插件内置。

## 仪式触发

只有 `HolyFather` 可以启动深红献祭仪式。

触发条件：

- 玩家属于深红阵营。
- 当前深红角色是 `HolyFather`。
- 游戏身份仍为 `Tutorial`。
- 使用的物品是 `SCP1576`。
- 当前房间是 `Hcz049` 或 `Hcz106`。
- 当前没有其他仪式正在进行。

触发时，插件会阻止 `SCP1576` 的原使用效果，将物品丢到地上，并启动仪式倒计时。

## 仪式过程

仪式开始后：

- 记录当前教皇作为仪式拥有者。
- 对深红阵营成员播放仪式音效。
- 全服显示仪式开始提示。
- 全服显示倒计时 HUD。

倒计时时长由 `RitualDurationSeconds` 控制。倒计时期间，所有玩家都不能拾取 `SCP1576`。

仪式会持续检查教皇状态。出现以下情况时，仪式立即中断：

- 教皇死亡。
- 教皇离开服务器。
- 教皇失去深红成员身份。
- 教皇不再是 `HolyFather`。
- 教皇不再保持 `Tutorial` 身份。
- 教皇离开有效仪式房间。
- 回合已经结束。

中断时会停止仪式音效、隐藏倒计时 HUD、清理仪式状态，并显示仪式中断提示。

## 仪式完成

倒计时结束且教皇仍然满足条件时，仪式完成。

仪式完成后，插件会击杀所有符合以下条件的玩家：

- 不属于深红阵营。
- 当前存活。
- 当前身份不是 `Tutorial`。

随后插件会调用游戏原生回合结束逻辑请求结束回合。由于 SCP:SL 的结算面板由原生逻辑控制，最终显示效果以服务器实际运行结果为准。

## 回合清理

回合结束、玩家离开、玩家死亡或深红玩家切换到非目标身份时，插件会清理相关深红状态。仪式中断和回合结束也会清理音频 session 与 HUD 倒计时，避免状态残留到后续回合。
