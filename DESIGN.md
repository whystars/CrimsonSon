# CrimsonSon 设计说明

这个文件只记录当前仓库应实现的玩法，不按更早那版 Word 稿里的旧机制走。

## 核心目标

深红之子的目标不是打赢正面团战，而是保护教皇完成仪式。

仪式在 `SCP-049` 或 `SCP-106` 收容室触发。完成后，深红阵营通过终结回合取得胜利表现。

## 回合开始

- 从 `ClassD` 中随机选出 1 名玩家
- 该玩家转为 `SCP-999-B`
- `SCP-999-B` 切换为 `Tutorial`
- 传送到配置项 `Spawn999BRoom` 对应房间

## 召唤阶段

`SCP-999-B` 进入广播室 `EzIntercom` 后触发召唤。

限制：

- 每回合只允许成功召唤一次
- 需要至少 `MinimumSpectatorsToSummon` 名观察者
- 最多拉起 `MaximumSummonCount` 名观察者

成功后：

- 全服显示提示
- 触发 Cassie 文案和音频
- 播放召唤音效
- 保底生成 `HolyFather`、`Heathen`、`Hallow`
- 其他名额按权重和角色上限分配

失败后：

- 如果当前 `SCP-999-B` 仍有效、仍在广播室、回合仍在继续，则允许再次尝试
- 否则本回合不能再召唤

## 仪式触发条件

只能由 `HolyFather` 触发。

条件：

- 玩家属于深红阵营
- 当前角色是 `HolyFather`
- 使用物品是 `SCP1576`
- 所在房间为 `Hcz049` 或 `Hcz106`

触发时：

- 阻止原物品使用
- 把 `SCP1576` 丢到地上
- 标记当前仪式房间
- 启动倒计时协程

## 仪式执行

协程开始后：

- 记录教皇玩家 ID
- 对深红阵营成员播放 `event.wav`
- 全服显示仪式开始提示
- 全服显示倒计时 HUD

倒计时期间持续检查：

- 教皇是否存活
- 教皇是否仍在正确房间
- 教皇是否仍属于深红阵营
- 教皇是否仍保持目标角色
- 回合是否已经结束

任一条件失效，仪式立即中断。

仪式期间：

- 所有人不能拾取 `SCP1576`

## 仪式完成

倒计时结束后：

- 杀死所有非深红阵营、且存活、且不是 `Tutorial` 的玩家
- 调用 `Round.End(true)` 结束回合

## 中断和清理

中断统一走状态清理逻辑。

需要清掉：

- 仪式中的音频 session
- 倒计时 HUD
- 仪式状态字段
- 全局提示残留

回合结束、玩家离开、玩家死亡、角色切换时，也需要保证状态被正确回收。
