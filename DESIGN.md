# CrimsonSon 玩法设计

本文档说明 CrimsonSon 当前版本的玩法规则和主要流程。它面向服务器管理者、测试人员和维护者，用来确认插件在回合中的预期行为。

当前设计以本文件和当前代码为准，不采用旧稿中的“5 分钟自动召唤”“电话可被夺取打断”等旧机制。

## 阵营目标

深红之子是第三方阵营。阵营目标不是常规正面对抗，而是保护 `HolyFather`（教皇）进入 `SCP-049` 或 `SCP-106` 收容室，使用 `SCP1576` 举行深红献祭仪式。

仪式完成后，插件会击杀非深红阵营的存活玩家，并请求游戏结束回合。最终结算面板由 SCP:SL 原生回合结算逻辑决定，当前插件不精确指定“深红阵营胜利队”。

## 回合开始

回合开始后，插件会从当前 D 级人员中随机选择一名玩家成为 `SCP-999-B`。

`SCP-999-B` 会被切换为 `Tutorial` 角色，并传送到配置项 `Spawn999BRoom` 指定的房间，默认是 `LczGlassroom`。

如果回合开始时没有可用 D 级人员，插件不会生成 `SCP-999-B`。

`SCP-999-B` 的任务是前往广播室并使用广播室话筒，借此召唤深红之子。

## 召唤触发

当前召唤不再通过“进入广播室后反复检测房间”触发。

正式触发方式是：

- 当前有效的 `SCP-999-B` 使用广播室话筒。
- 广播室话筒状态必须是 `Ready` 或 `Starting`。
- 回合仍在进行。
- 本回合还没有成功召唤过深红之子。

这样可以让原版广播室话筒冷却自然限制短时间重复尝试。

如果 `SCP-999-B` 使用话筒时观察者人数不足，召唤失败，并允许之后在话筒冷却结束后再次尝试。只要本回合没有成功召唤过，且当前 `SCP-999-B` 仍然有效，就可以继续重试。

## 召唤候选

召唤候选必须是旁观状态。

真实玩家候选条件：

- 玩家存在。
- 玩家身份是 `Spectator`。

dummy NPC 候选条件：

- 配置项 `CountDummySpectatorsForSummon` 为 `true`。
- dummy 存在于 `Player.DummyList` 或 `Player.List`。
- dummy 身份是 `Spectator`。

插件会合并 `Player.List` 和 `Player.DummyList`，并按 `PlayerId` 去重，避免同一个 dummy 被重复计数。

`CountDummySpectatorsForSummon` 主要用于测试服人数不足时跑完整流程。正式服如果不希望 dummy 参与召唤候选，可以关闭该配置。

## 召唤限制

召唤限制：

- 每回合只允许成功召唤一次。
- 候选人数必须达到 `MinimumSpectatorsToSummon`。
- 单次最多召唤 `MaximumSummonCount` 名候选。

召唤成功后会从候选池中随机选人生成深红阵营成员。

优先保底生成：

- `HolyFather`
- `Heathen`
- `Hallow`

剩余名额按角色权重分配，并受每个角色人数上限限制。

## 召唤表现

召唤成功后：

- 全服显示召唤成功提示。
- 全服显示 CASSIE 字幕。
- 全体玩家播放 `cassie.wav`。
- 生成深红阵营成员。
- 深红阵营成员播放 `summon.wav` 入场音乐。

CASSIE 不读中文文案。插件发送的 CASSIE message 只包含英文句号，不包含可读单词；玩家看到的自定义字幕来自 `translations.yml` 中的 `CassieBroadcast`，真正的声音来自 `cassie.wav`。

字幕显示时间由 `CassieBroadcastHoldSeconds` 控制。插件会让 CASSIE message 发送对应数量、用空格分隔的英文句号，例如 `. . . .`；实测约 1 个 `.` 延长 1 秒显示时间。这些句号不会追加到中文字幕后面。

## 音频规则

当前音频资源：

- `Audio/cassie.wav`
- `Audio/summon.wav`
- `Audio/event.wav`

当前可听范围：

| 音频 | 触发时机 | 可听范围 |
|---|---|---|
| `cassie.wav` | 深红之子召唤成功时 | 全体玩家 |
| `summon.wav` | 深红成员生成完成后 | 深红阵营成员 |
| `event.wav` | 仪式开始时 | 全体玩家 |

音频通过 `SCPSL-AudioManagerAPI` session 播放。仪式中断、回合结束或插件卸载时会停止相关 session，避免声音残留。

## 角色体系

深红阵营角色如下：

| ID | 角色 | 说明 |
|---|---|---|
| 0 | `SCP999B` | 开局特殊单位，使用广播室话筒触发召唤。 |
| 1 | `HolyFather` | 教皇，唯一可以启动仪式的核心角色。 |
| 2 | `Heathen` | 深红成员，负责保护教皇。 |
| 3 | `Hallow` | 深红成员，负责保护教皇。 |
| 4 | `Follower` | 深红成员，负责保护教皇。 |
| 5 | `EvilFollower` | 深红成员，负责保护教皇。 |
| 6 | `Fanatic` | 深红成员，负责保护教皇。 |

所有深红角色在游戏身份上都会使用 `Tutorial`。角色名称、介绍文本、人数上限、分配权重和血量可以通过配置或翻译文件调整。

默认物品列表由插件代码内置，当前不在配置文件中管理。

## 仪式触发

只有 `HolyFather` 可以启动深红献祭仪式。

触发条件：

- 玩家属于深红阵营。
- 当前深红角色是 `HolyFather`。
- 游戏身份仍为 `Tutorial`。
- 使用的物品是 `SCP1576`。
- 当前房间是 `Hcz049` 或 `Hcz106`。
- 当前没有其他仪式正在进行。

触发时，插件会阻止 `SCP1576` 的原使用效果，将物品丢到地上，记录这个掉落物作为仪式召唤物，并启动仪式倒计时。

## 仪式过程

仪式开始后：

- 记录当前教皇作为仪式拥有者。
- 全体玩家播放 `event.wav`。
- 全服显示仪式开始提示。
- 全服显示倒计时 HUD。

倒计时时长由 `RitualDurationSeconds` 控制。

倒计时期间，深红阵营成员不能拾取作为仪式召唤物的 `SCP1576`。其他阵营玩家如果成功拾取这个 `SCP1576`，仪式立即中断。

地图上的其他 `SCP1576` 不受这条规则影响。

仪式会持续检查教皇状态。出现以下情况时，仪式立即中断：

- 教皇死亡。
- 教皇离开服务器。
- 教皇失去深红成员身份。
- 教皇不再是 `HolyFather`。
- 教皇不再保持 `Tutorial` 身份。
- 回合已经结束。

中断时会停止仪式音效、隐藏倒计时 HUD、清理仪式状态，并显示仪式中断提示。插件会通过仪式序号校验让旧倒计时协程退出，避免仪式已中断后继续静默倒计时、击杀玩家或结束回合。

## 仪式完成

倒计时结束且教皇仍然满足条件时，仪式完成。

仪式完成后，插件会击杀所有符合以下条件的玩家：

- 不属于深红阵营。
- 当前存活。
- 当前身份不是 `Tutorial`。

随后插件会调用游戏原生回合结束逻辑请求结束回合。

由于 SCP:SL 的结算面板由原生逻辑控制，最终显示效果以服务器实际运行结果为准。

## 回合清理

以下情况会清理相关状态：

- 回合结束。
- 玩家离开服务器。
- 玩家死亡。
- 深红玩家切换到非目标身份。
- 仪式中断。
- 插件卸载。

清理内容包括：

- 深红成员状态。
- 自定义信息。
- HUD 显示。
- 仪式状态。
- 音频 session。

## 配置项概要

常用配置项：

| 配置项 | 作用 |
|---|---|
| `MinimumSpectatorsToSummon` | 召唤所需最少候选人数 |
| `MaximumSummonCount` | 单次最多召唤人数 |
| `CountDummySpectatorsForSummon` | 是否允许旁观状态 dummy 计入召唤候选 |
| `RitualDurationSeconds` | 仪式持续时间 |
| `SummonAudioVolume` | `summon.wav` 音量 |
| `EventAudioVolume` | `event.wav` 音量 |
| `CassieAudioVolume` | `cassie.wav` 音量 |
| `CassieBroadcastHoldSeconds` | CASSIE message 自动发送的英文句号数量 |
| `SpawnRoom` | 深红阵营生成房间 |
| `Spawn999BRoom` | `SCP-999-B` 开局房间 |
| HUD 坐标和字号 | 控制角色介绍、通知、倒计时 HUD |
| `Roles` | 各深红角色人数上限、权重、血量 |

升级配置结构时，建议删除旧 `config.yml` 和 `translations.yml` 重新生成，再按新字段调整。

## 翻译项概要

常用翻译项：

- 命令提示。
- 召唤成功提示。
- 观察者不足提示。
- 无法再次尝试召唤提示。
- 仪式开始提示。
- 仪式中断提示。
- CASSIE 字幕文案。
- 倒计时 HUD 文案。
- 终局击杀原因。
- 角色名和角色描述。

角色描述支持 `{Name}` 占位符。

## 测试重点

测试时优先确认：

- 回合开始是否能正常选出 `SCP-999-B`。
- `SCP-999-B` 是否能切为 `Tutorial` 并传送到 `Spawn999BRoom`。
- 999-B 使用广播室话筒时是否触发召唤日志。
- 观察者不足时是否失败并允许话筒冷却后重试。
- dummy 旁观候选是否受 `CountDummySpectatorsForSummon` 控制。
- 召唤成功后保底角色是否正常生成。
- `cassie.wav` 是否全体可听。
- CASSIE 是否不读字、只显示字幕。
- CASSIE 字幕显示时间是否足够。
- `summon.wav` 是否只有深红阵营成员可听。
- `event.wav` 是否全体可听。
- 教皇在 049/106 使用 `SCP1576` 是否启动仪式。
- 仪式期间深红成员是否无法拾取仪式 `SCP1576`。
- 非深红成员成功拾取仪式 `SCP1576` 后是否中断仪式。
- 仪式中断时 HUD、音频和状态是否清理干净。
- 仪式成功后是否正确处理非深红玩家并结束回合。
